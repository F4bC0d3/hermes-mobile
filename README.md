# 🪽 Hermes Mobile

> Native Android app for self-hosted Hermes Agent. Point it at your HF Space, log in once with `GATEWAY_TOKEN`, and chat with your Hermes Agent through a ChatGPT/Claude-style mobile interface.

**This is not a fresh AI assistant.** It's a thin native wrapper around three existing open-source projects, sized and styled for an Android phone:

- **[Hermes Agent](https://github.com/NousResearch/hermes-agent)** by **[Nous Research](https://nousresearch.com)** — the AI agent itself: persistent memory, tool execution, scheduled jobs, multi-provider LLM support, skills, profiles. All actual intelligence is theirs.
- **[Hermes WebUI](https://github.com/nesquena/hermes-webui)** by **[@nesquena](https://github.com/nesquena)** — the chat surface inside the WebView. SSE streaming, slash commands, themes, mermaid diagrams, file browser, voice input, all 100+ features. Used as-is.
- **[HuggingMes](https://github.com/somratpro/HuggingMes)** by **[@somratpro](https://github.com/somratpro)** — the HF Space packaging that lets you run all this for free in the cloud, with HF Dataset persistence and Cloudflare keep-alive.

This app provides a native shell: an encrypted login screen, a Material 3 top bar, a sessions drawer fetched from the WebUI's API, a bottom-sheet menu, and the chat surface itself loaded as a WebView. That's it.

The matching Space deployment lives at **[F4bC0d3/huggingmes-hermes-webui](https://github.com/F4bC0d3/huggingmes-hermes-webui)**.

## Prerequisites

You need a running Hermes WebUI instance to point the app at. The easiest way is to duplicate my live demo Space:

1. Go to [huggingface.co/spaces/f4b404/hermes](https://huggingface.co/spaces/f4b404/hermes) → **⋮** → **Duplicate this Space**
2. In Space Settings → Variables and secrets, set `GATEWAY_TOKEN`, `LLM_API_KEY`, `LLM_MODEL`, and (recommended) `HF_TOKEN`
3. Wait for the build, then your Space URL is `https://<you>-<name>.hf.space`

Full instructions: [F4bC0d3/huggingmes-hermes-webui](https://github.com/F4bC0d3/huggingmes-hermes-webui#setting-up-your-own-space).

## Get the APK

Two paths.

### Option 1: Download a pre-built APK from GitHub Releases

Go to the [Releases tab](https://github.com/F4bC0d3/hermes-mobile/releases) of this repo and grab the latest `hermes-mobile-debug.apk`.

If there's no release yet (or you want the absolute latest commit), open [Actions](https://github.com/F4bC0d3/hermes-mobile/actions) → click the latest **Build Android APK** run → scroll to **Artifacts** → download `hermes-mobile-debug-apk`.

### Option 2: Build it yourself

If you have Android Studio Hedgehog or newer:

```bash
git clone https://github.com/F4bC0d3/hermes-mobile.git
cd hermes-mobile
./gradlew :app:assembleDebug
# APK lands in app/build/outputs/apk/debug/app-debug.apk
```

The Gradle wrapper is generated by Studio on first sync. Headless builds:

```bash
gradle wrapper --gradle-version 8.10.2 --distribution-type bin
./gradlew :app:assembleDebug
```

Requires JDK 17 (Temurin recommended) and the Android SDK with platform 34 + build-tools 34.0.0.

## Sideload onto your phone

The APK is unsigned (debug build) but Android happily installs it for personal use.

1. Transfer `hermes-mobile-debug.apk` to your phone (drive, AirDrop, KDE Connect, USB cable, anything)
2. Open it from your file manager
3. When Android warns "Install unknown apps", allow it for your file manager / browser this once
4. The app installs as **Hermes**

If using `adb` over USB:

```bash
adb install -r hermes-mobile-debug.apk
```

## First run

1. Launch **Hermes** from your app drawer
2. **Space URL**: paste your `*.hf.space` URL (e.g. `https://f4b404-hermes.hf.space` for my demo)
3. **GATEWAY_TOKEN**: same token you set on your Space
4. Tap **Continue**

The app calls `/login`, captures the session cookie, and loads the chat UI. Cookie + URL + token are stored encrypted via Android Keystore (AES-256-GCM through `EncryptedSharedPreferences`).

## What the app gives you

| Native (Compose Material 3) | WebView (hermes-webui as-is) |
| --- | --- |
| Login screen with `/login` cookie capture | Chat composer + SSE streaming + tool cards |
| Encrypted credential storage (Android Keystore) | Slash commands, profiles, themes, skills |
| Top app bar (title + new chat + ⋮ menu) | Mermaid, syntax highlighting, code copy |
| Sessions drawer (live from `/api/sessions`) | Cron tasks, memory browser, voice input |
| Bottom-sheet menu (theme, dashboard, sign out) | Workspace file browser, file uploads |
| Splash + adaptive icon + share-to-Hermes target | Everything else hermes-webui ships |

The split is intentional: zero re-implementation of the chat itself so every Hermes WebUI release just keeps working.

## Architecture in one paragraph

Single-Activity Compose app, Material 3, edge-to-edge. `MainActivity` swaps between `LoginScreen` and `ChatShell` based on whether credentials exist in `SessionStore` (an `EncryptedSharedPreferences` wrapper). `ChatShell` is a top app bar + `ModalNavigationDrawer` + WebView. The drawer pulls sessions from `/api/sessions` via OkHttp. Tapping a session loads `/session/<id>` in the WebView. The WebView's cookie store is seeded with the captured `webui_session` cookie at launch, so every request inside it is authenticated. Permissions for mic + camera are passed through to the WebView so voice input works. ProGuard keeps for kotlinx-serialization companions and `@JavascriptInterface` members.

## Security notes

- Cleartext HTTP blocked except for `localhost`, `127.0.0.1`, and `10.0.2.2` (emulator host loopback)
- Tokens, URLs, and cookies stored only in `EncryptedSharedPreferences` (`hermes_session.xml`)
- The `webui_session` cookie is HttpOnly so JS in the WebView can't read it; only Android's `CookieManager` can
- `android:fullBackupContent` and `android:dataExtractionRules` exclude credentials from cloud backups and device transfer — re-login required after a fresh install
- ProGuard shrinks/obfuscates everything except serializer companions and `@JavascriptInterface` methods

## What's not here yet

- **FCM push notifications** for cron job completions and background agent errors. Easy to add — needs a small webhook endpoint on the Space side
- **Biometric unlock** before showing the WebView (`androidx.biometric` is a few-line addition)
- **Foreground service** for cron alerts so the app can fire notifications even when closed
- **Background sync** to keep the sessions drawer warm

## Why not just install the Space as a PWA?

You absolutely can — hermes-webui ships `sw.js` and is mobile-responsive. The native shell adds:

1. A real Android app icon and splash, no "Add to Home Screen" detour
2. Share-to-Hermes intent target so other apps can pipe text/images straight in
3. Encrypted credential persistence that PWAs can't do
4. Crash-resistance: if the WebView dies, native chrome keeps the app alive
5. Future hooks for push notifications and biometric unlock

If none of that matters to you, the PWA route is one tap. This shell is for when it does.
